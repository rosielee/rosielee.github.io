  • Browsersync
  • browsersync.io

  • Instructions/Info

    setting vagrant up with browser sync

    • install node
    • install browsersync: sudo npm install -g browser-sync
    • navigate to the css folder of the website (e.g. cd ~/scotchbox/dev.infiniteobjects.co.uk/public/wp-content/themes/infiniteobjects/assets/css) (NB watch out for public, may or may not be in this directory)
    • start browser sync: browser-sync start --proxy "dev.infiniteobjects.co.uk" --files "compiled/*.css"
    • the website will open in a browser at localhost:3000
    • use network utility find your own computer's IP (e.g. 10.0.0.13)
    • navigate to your computer's IP address with the port extension (e.g. 10.0.0.13:3000)

    all changes made locally on your mac, not in virtual machine (i.e. vagrant)


    NOTE safari can dev tool an iOS device (connected by cable). iOS tends to be the problem child anyhow; android can be mostly tested in browser in my experience

  • Trello
  • trello.com

  • Instructions/Info

    structure -

    digital boards are named: `DIGITAL [job number] [job name]`

    • each board has four columns - inbox, todo, doing, done
    • boards are starting to have a fifth column between inbox and todo called later - this is for features that are not to be included in this iteration of work, but in the next iteration instead (iteration == sprint in agile terminology)
    • other trello boards may have other layouts

    labels -

    • feature / improvement - the site could potentially go live without it
    • copy / content - all placeholder text to be replaced
    • client services - i.e. something that's not design or development
    • bug / snags - will probably need fixing before going live - snags could potentially wait
    • design - studio's involvement required before development)
    • development - code!
