  • Browsersync
  • browsersync.io

  • Instructions/Info

    setting vagrant up with browser sync

    • install node
    • install browsersync: sudo npm install -g browser-sync
    • navigate to the css folder of the website (e.g. cd ~/scotchbox/dev.infiniteobjects.co.uk/public/wp-content/themes/infiniteobjects/assets/css) (NB watch out for public, may or may not be in this directory)
    • start browser sync: browser-sync start --proxy "dev.infiniteobjects.co.uk" --files "compiled/*.css"
    • the website will open in a browser at localhost:3000
    • use network utility find your own computer's IP (e.g. 10.0.0.13)
    • navigate to your computer's IP address with the port extension (e.g. 10.0.0.13:3000)

    all changes made locally on your mac, not in virtual machine (i.e. vagrant)


    NOTE safari can dev tool an iOS device (connected by cable). iOS tends to be the problem child anyhow; android can be mostly tested in browser in my experience

  • SSH

  • Instructions/Info

    we don't actually have FTP set up on our server instead we can use SSH to log into the serverthis does mean however that the whole server (computer running linux) is exposed - this wouldn't be the case with a standard host (e.g. 1and1, etc) so extreme caution is advised!

    • a new user is created on each server we work from
    • this user must then su - and enter the root password to become root
    • once the user has root access, sites can be deployed via git
  • Vagrant
  • vagrantup.com

  • Vagrant setup - once set up in order to create a new domain one must...

    • open the Vagrant file
    • add a new domain to the DOMAIN array
    • navigate to that file in terminal
    • in that folder, enter vagrant provision - this will create the folders and restart apache, etc
    • open /etc/hosts and enter a new 'DNS' record at the bottom *
    • navigate to the new domain in the browser
    • development - code!

    * to open the host file I'd...

    • in sublime cmd + o
    • 'go to folder with' cmd + shift + g
    • type /etc and press enter
    • search for hosts file with cmd + p

  • Trello
  • trello.com

  • Instructions/Info

    structure -

    digital boards are named: `DIGITAL [job number] [job name]`

    • each board has four columns - inbox, todo, doing, done
    • boards are starting to have a fifth column between inbox and todo called later - this is for features that are not to be included in this iteration of work, but in the next iteration instead (iteration == sprint in agile terminology)
    • other trello boards may have other layouts

    labels -

    • feature / improvement - the site could potentially go live without it
    • copy / content - all placeholder text to be replaced
    • client services - i.e. something that's not design or development
    • bug / snags - will probably need fixing before going live - snags could potentially wait
    • design - studio's involvement required before development)
    • development - code!
  • Sass

  • Instructions/Info

    Starting SASS

    SASS comes installed on most machines (check with sass -v). For each dev site you will need to start sass for so:

    • Navigated to CSS folder of the project by: cmd line> cd folder path (not file path)> ls -la(to check ok)
    • The preprocessing of SASS files into CSS is best (and quickest) done on the command line with the following command: sass --watch sass:compiled

  • Security notes

  • Instructions/Info

    • change the name of the wp-content folder 
      define('WP_CONTENT_DIR', dirname(__FILE__) . '/content'); define('WP_CONTENT_URL', '/content');
    • stop system files being directly accessed 
      if ( ! defined( 'ABSPATH' ) ) { exit; // Exit if accessed directly }
    • add this to all .htaccess's 
      # Block the include-only files.  RewriteEngine On RewriteBase / RewriteRule ^wp-admin/includes/ - [F,L] RewriteRule !^wp-includes/ - [S=3] RewriteRule ^wp-includes/[^/]+\.php$ - [F,L] RewriteRule ^wp-includes/js/tinymce/langs/.+\.php - [F,L] RewriteRule ^wp-includes/theme-compat/ - [F,L]
